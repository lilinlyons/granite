Event-Driven Architecture.

About

Event-driven architecture (EDA) is a software design paradigm where system behaviour is driven by the detection and handling of events. EDA enables loose coupling of applications by introducing a middleman known as an event broker. This architecture additionally relies on event producers, which generate events, and event consumers, which respond to those events. Key technologies that support EDA include Apache Kafka, RabbitMQ, and Amazon SNS/SQS, which provide robust event streaming and messaging capabilities. EDA is particularly useful in real-life applications requiring real-time responsiveness and high scalability, such as financial trading systems where milliseconds matter and microservices architectures where services need to communicate asynchronously and efficiently. Fundamentally, EDA recognises a meaningful change and thus distributes the data to all associated systems. EDA does not necessarily consist of one programming language; it is rather a model or architecture to efficiently aid the transmission and detection of changes.

Key Components

Producers

These are components that generate events. An event can be any significant change in state or condition, such as a user action, a message from another system, or a sensor reading. In the Hedge-Fund industry, these could be a trade that is executed or the receipt of real-time market data. The basic attributes of an event include time, source, key, header, metadata, and payload. The producer does not need to be aware of other components in the EDA system.

Brokers

Event brokers sit between producers and consumers; it is a direct information pipeline between the event generator and consumers. Brokers are a type of middleware that can be a hardware appliance, software, or software-as-a-service. Brokers store messages as channels allowing multiple consumers to read the same message.

Consumers

Event consumers are components that respond to events. They perform actions based on the events they receive. Consumers can also be producers if their actions generate new events.
